Why I built this

Hunting for remote homologues of a phage protein usually means stitching together a pipeline by hand: build an alignment, make an HMM, search a database, pull hits, validate ORFs, feed the good ones back in, repeat. Every step is a different tool with its own flags, and by the third iteration you have lost track of exactly what you ran.

I wanted that whole loop to be one command that produces the same answer every time. So I packaged the six-frame search, ORF validation, and iterative convergence into a single reproducible run - with golden-file regression tests so I know a code change did not silently change the biology.

The point is reproducibility: one command, a recorded configuration, and regression tests - so a homologue search you run today can be re-run and trusted months later.

What it does

🧬

Six-frame search

Searches nucleotide databases in all six frames so you do not miss homologues hiding in unannotated regions.

✅

ORF validation

Filters hits down to real open reading frames rather than spurious matches.

🔁

Iterative convergence

Feeds validated hits back into the profile and re-searches until the family stops growing.

🧪

Golden-file tests

Regression tests pin the expected output, so refactors cannot quietly change results.

How it works

  1. Give it a seed

    Start from a protein family or a set of sequences you care about.

  2. Run the command

    It builds the HMM, searches the databases in six frames, and validates the ORFs it finds.

  3. Converge and export

    Validated hits feed back in until the search converges; results and provenance are exported for you.

Reproducible by design

Bioinformatics results are only as good as your ability to reproduce them. This tool ships a locked environment, a citation file, and a golden-file test harness - so the same input gives the same output, and reviewers (or future me) can verify it.
